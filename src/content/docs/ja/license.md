title: ライセンス FAQ
description: Dibao の BUSL-1.1、fair-code、Change Date、商用利用の境界。
---

Dibao は Business Source License 1.1（BUSL-1.1）を使用し、Additional Use Grant によって source-available / fair-code / delayed open source の形を取ります。各公開リリースは、それぞれの Change Date 後に Apache License 2.0 へ自動的に変わります。

## 現在のライセンスは？

現在のバージョンは BUSL-1.1 です。Change License は Apache-2.0 です。

## fair-code ですか？

はい。Dibao は fair-code、source-available、セルフホスト可能なプロジェクトです。BUSL-1.1 はそのための法的な仕組みです。

## オープンソースですか？

Change Date 前の現在バージョンは OSI の意味でのオープンソースではありません。Change Date 後、その具体的なリリースは Apache-2.0 になります。

## 自分でホストできますか？

はい。個人、家庭、研究、評価、学習、非商用用途では無料でセルフホストして使えます。

## 会社内で使えますか？

はい。会社、学校、非営利団体などは、内部利用として Dibao を無料で使用、変更、セルフホスト、本番利用できます。

## 顧客向けに有料でデプロイできますか？

できます。ただし、インスタンスは顧客自身の環境、アカウント、または顧客が管理するインフラ上で動く必要があります。Dibao を自分のホスティング平台、ホワイトラベル製品、再販売製品、競合 SaaS にすることは、別途商用許可なしにはできません。

## 有料 SaaS やホワイトラベル版を提供できますか？

できません。Licensor から個別の商用許可を得る必要があります。問い合わせ先は https://dibao.app です。

## 4 年後には何が起きますか？

その具体的なリリースが Change Date に Apache-2.0 へ自動変換されます。

## main ブランチはどう扱いますか？

`main` は開発ブランチであり、TODO 日付や未リリースのパラメータを含むことがあります。Release Date、Change Date、法的スナップショットは各 release tag の `LICENSE.md` で固定されます。
